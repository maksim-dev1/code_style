# Changelog

Заметные изменения гайда. Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версии — [SemVer](https://semver.org/lang/ru/).

## [1.1.1] — 2026-06-26

### Changed
- **`retrofit` (`@RestApi`) — основной способ описывать api-сервис** (§12.2): `abstract`-интерфейс
  с аннотациями `@GET/@POST/@PATCH`, `@Path/@Query/@Body`, `part '*.g.dart'`, методы возвращают
  DTO; реализация — кодоген. Ручной вызов `dio` переведён в §12.8 как исключение для единичных
  случаев. §12.4 (envelope): распаковка для retrofit — в интерсепторе `onResponse`.
- В таблицу стека (§1) добавлены `dio` + `retrofit` (сеть, когда есть бэкенд).

## [1.1.0] — 2026-06-26

### Changed
- **§12 «Сеть / API» переписан в полноценный раздел** по работе с интернетом (`dio`),
  разбит на подразделы:
  - **12.1** один настроенный `Dio` в DI (baseUrl/таймауты/интерсепторы), таймауты обязательны;
  - **12.2** api-сервис (data-provider) → DTO, тело через `toJson()`, один метод = один эндпоинт;
  - **12.3** интерсепторы: токен авторизации, refresh на 401 (синхронизация параллельных,
    ротация refresh-токена), логирование `talker_dio_logger` последним;
  - **12.4** распаковка envelope `{success, data}` одним хелпером;
  - **12.5** `DioException` → доменная ошибка по `DioExceptionType` (тайм-аут/нет связи/4xx/5xx);
  - **12.6** отмена запроса `CancelToken` (+ `restartable()` для живого поиска);
  - **12.7** повторы только для идемпотентных/сетевых сбоев, файлы (`FormData`/прогресс);
  - **12.8** типизированный клиент `retrofit` (опционально);
  - **12.9** живые данные — **WebSocket / SSE / polling** как `Stream<T>`: один канал на
    ресурс, авто-reconnect с бэкоффом, реакция на жизненный цикл, блок — тонкий подписчик.

### Added
- §27 (частые ошибки): пункт про сеть — `Dio()` на каждый запрос / без таймаутов /
  `e.toString()` пользователю / парсинг `resp.data` в виджете.

## [1.0.0] — 2026-06-19

### Added
- Первый полный гайд `CODE_STYLE_GUIDE.md` (Части I–IV, разделы 0–27).
  - **Основы:** философия, стек (BLoC/Freezed/Drift), старт проекта с нуля, точка входа
    `main`, три слоя, дробление на мелкие фичи.
  - **Рецепт фичи:** entity → DTO/таблица БД → mapper (`abstract class` со `static`
    `toEntity`/`toDto`/`fromRow`) → репозиторий (порт + impl) → BLoC → provider → экран.
  - **BLoC:** события и состояния на `@freezed sealed` (состояние = union фаз, без флагов),
    один `on<>`+`switch`; DI именованными `required`-параметрами + список инициализации.
  - **UI:** тонкий коннектор (`BlocBuilder`) ↔ чистые UI-виджеты; выбор виджетов
    (`InkWell`/`ColoredBox`/`ListView.builder`…); дизайн-токены вместо хардкода.
  - **Углублённо:** сервисы, навигация, сеть/API, обработка ошибок (`Failure`), кэш/оффлайн
    (SSOT), event transformers, окружения, формы, состояния экрана, списки, темизация,
    локализация, адаптивность, тесты, надёжность, инструменты.
  - **Шпаргалки:** чек-лист новой фичи, частые ошибки новичков.
- `README.md` с содержанием и инструкцией по использованию в проектах.
