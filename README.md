# code_style — гайд по архитектуре Flutter-проектов

Единый **источник правды** по стилю и архитектуре кода для Flutter-проектов
(стек **BLoC + Freezed + Drift**). Один гайд — для всех проектов: правила живут здесь
и обновляются здесь, а проекты ссылаются на него.

## 📖 Гайд

→ **[CODE_STYLE_GUIDE.md](CODE_STYLE_GUIDE.md)** — пошаговое руководство «с нуля», для
начинающих: каждое правило с объяснением **зачем**, примерами и чек-листами.

**Содержание**

- **Часть I. Основы** — философия · технологии (стек) · поднять проект с нуля · точка
  входа `main` · три слоя (`domain`/`data`/`presentation`) · одна фича = одна забота.
- **Часть II. Рецепт фичи и базовые правила** — entity → DTO/таблица → **mapper**
  (`abstract class`, `toEntity`/`toDto`/`fromRow`) → репозиторий → **BLoC** → provider →
  экран · DI (именованные `required`) · **UI** (коннектор ↔ чистые виджеты) · токены ·
  логирование · правила написания кода.
- **Часть III. Углублённые темы** — навигация · **сеть/API** (один `Dio` в DI · api-сервис
  на `retrofit`/`@RestApi` → DTO · интерсепторы токена/refresh/лога · распаковка envelope ·
  `DioException` → доменная ошибка · отмена · файлы · **WebSocket/SSE/polling**) · обработка
  ошибок · кэш/оффлайн (SSOT) · event transformers · окружения · формы · состояния экрана ·
  списки · темизация · локализация · адаптивность · тесты · надёжность · инструменты.
- **Часть IV. Шпаргалки** — чек-лист новой фичи · частые ошибки новичков.

## 🚀 Как использовать в проекте

Не копируй гайд в проект — **ссылайся** на него. В `CLAUDE.md` проекта добавь секцию:

```markdown
## 📐 Код-стайл и архитектура — ВНЕШНИЙ ГАЙД (источник правды)

Канонические правила написания кода — https://github.com/maksim-dev1/code_style.git
Следуй ему при написании и рефакторинге любого кода.
```

Так Claude Code и новые разработчики знают, где искать конвенции, а правила не
расходятся между проектами.

> Имена хелперов в гайде условные (`AppLogger`, `AppColors`, …) — в конкретном проекте
> это свои токены/сервисы (например `BeaconLogger`, `context.beaconColors`). Важны
> **паттерны**, а не имена.

## ✍️ Обновление

Меняешь конвенцию → правишь `CODE_STYLE_GUIDE.md` → коммит с записью в
[CHANGELOG.md](CHANGELOG.md). Проекты подтягивают актуальную версию по ссылке.
